# 08_DAILY_WORKFLOW

## Rutina de Desarrollo

1. **Revisión de Tareas (`task.md`)**:
   - Antes de empezar, marcar `[/]` en la tarea activa.
   - Al terminar, marcar `[x]` y asegurar que los criterios de aceptación (`06_ACCEPTANCE_CRITERIA.md`) se cumplen.

2. **Planificación Obligatoria (`PLAN.md` / `implementation_plan.md`)**:
   - **Antes de escribir código**: Generar/Actualizar el plan de implementación.
   - Detallar archivos a tocar y estrategia.
   - Esperar confirmación del usuario.

3. **Ejecución y Verificación**:
   - Implementar cambios siguiendo las reglas (`07_IMPLEMENTATION_RULES.md`).
   - **Visual QA**: Generar capturas de pantalla de los cambios de UI.
   - **Tests**: Ejecutar tests relevantes si aplican.

4. **Commits y Mensajes**:
   - Mensajes claros y descriptivos.
   - Referenciar ID de tarea si aplica.
